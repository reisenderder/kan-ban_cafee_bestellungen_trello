# Правила `.agents`

Эти правила действуют для материалов и инструкций AI-агентов.

1. Агент сначала читает корневой `README.md`, затем правила нужной папки, затем рабочие документы.
2. Агент не создает код, планы или спецификации без связи с утвержденными документами.
3. Агент не меняет смысл Global Spec без отдельного явного утверждения.
4. Агент пишет проектные выводы и документацию на русском языке.
5. Агент должен кратко фиксировать, какие файлы изменены и почему.
6. Если задача неясна или может нарушить правила проекта, агент сначала уточняет решение.
7. После сообщения пользователя `pull request смержен` агент выполняет синхронизацию с `main`, а следующую задачу начинает с новой ветки от актуального `main`.
8. При завершении плана агент до Git-публикации обновляет соответствующие хронологические журналы в `roadmap/`.
