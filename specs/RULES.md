# Правила `specs`

Эти правила действуют для всех документов внутри `specs/`.

1. Все спецификации пишутся на русском языке.
2. Документы развиваются сверху вниз: `Global Spec -> Functional Map -> Feature Specs -> Technical Specs -> User Stories -> Work Plans`.
3. Global Spec фиксирует смысл, границы, роли и фундаментальные принципы.
4. Functional Map раскрывает возможности, участников, пути, ситуации и MVP-границы без технической реализации.
5. Feature Specs описывают отдельные функции только после связи с Global Spec и Functional Map.
6. Technical Specs описывают реализацию, но не расширяют бизнес-смысл функции.
7. User Stories проверяют сценарии, отклонения и критерии приемки.
8. Work Plans создаются только после утвержденных спецификаций и user stories.
9. Открытые вопросы должны быть явно отмечены и не должны скрываться внутри текста.
10. Если документ нижнего уровня противоречит документу верхнего уровня, приоритет имеет верхний уровень до отдельного утверждения изменения.
