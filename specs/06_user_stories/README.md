# User Stories

## Назначение раздела

Этот раздел хранит пользовательские истории, сценарии и критерии приемки после Global Spec, Functional Map, Feature Specs и Technical Specs.

User Stories отвечают на вопрос: как конкретный участник системы проходит путь, какой результат он должен получить и какие отклонения система обязана обработать.

## Правила раздела

1. Каждая история должна ссылаться на утвержденные или текущие спецификации верхнего уровня.
2. User Stories не выбирают фреймворк, базу данных, API или инфраструктуру.
3. User Stories должны проверять happy path, отклонения, роли, видимость данных и критерии приемки.
4. Если открытый вопрос можно закрыть без противоречия верхним документам, решение фиксируется в разделе `Закрытые решения`.
5. Если вопрос нельзя закрыть без изменения Global Spec или Functional Map, он остается в разделе `Открытые вопросы`.
