# Feature Specs

## Назначение раздела

Этот раздел хранит спецификации отдельных функций продукта после Global Spec и Functional Map.

Feature Specs отвечают на вопрос: как должна вести себя конкретная функция, где ее границы и какие сценарии она должна покрывать.

## Правила раздела

1. Каждый Feature Spec должен ссылаться на Global Spec и Functional Map.
2. Feature Spec описывает поведение функции, границы, сценарии, состояния и открытые вопросы.
3. Feature Spec не описывает базу данных, API, фреймворки и финальную техническую реализацию.
4. Если функция требует технического решения, оно выносится в Technical Spec.
5. Если функция требует проверки пути пользователя, она выносится в User Stories.

## Текущие документы

1. `Feature_Order_Entry.md` - веб-витрина, корзина и оформление заказа.
2. `Feature_Contact_Confirmation.md` - подтверждение Telegram и Email.
3. `Feature_Menu_Management.md` - управление меню и доступностью блюд.
4. `Feature_Order_CRM.md` - CRM менеджера и обработка заказа.
5. `Feature_Courier_Delivery.md` - курьер, доставка и подтверждение доставки.
6. `Feature_Resolution_Department.md` - отдел урегулирования, желтый и красный список.
7. `Feature_Complaints.md` - жалобы клиента и результат обработки.
8. `Feature_Returned_Order_Discount.md` - возвращенный заказ и повторная продажа со скидкой.

В этот раздел добавляются отдельные Feature Specs для функций, уже связанных с Global Spec и Functional Map.
