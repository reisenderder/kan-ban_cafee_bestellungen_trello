# Technical Specs

## Назначение раздела

Этот раздел хранит технические спецификации проекта после Global Spec, Functional Map и Feature Specs.

Technical Specs отвечают на вопрос: как реализовать утвержденное поведение системы без изменения бизнес-смысла, ролей и границ, уже закрепленных в верхних документах.

## Правила раздела

1. Technical Spec не должен добавлять новую бизнес-функцию, если она не описана в Feature Specs.
2. Technical Spec должен ссылаться на Global Spec, Functional Map и связанные Feature Specs.
3. Technical Spec может описывать роли, права, данные, состояния, проверки, интеграции, ограничения, журналы и технические правила.
4. Technical Spec не должен выбирать финальный фреймворк, библиотеку или инфраструктуру без отдельного решения.
5. Открытые технические вопросы должны быть вынесены в отдельный раздел документа.

## Текущие документы

1. `Technical_Access_Audit.md` - роли, права доступа, маскирование данных, аудит раскрытия клиентских данных и правила совпадения признаков риска.
2. `Technical_Admin_Complaints.md` - администрирование, жалобы, журналы и ограничения прав.
3. `Technical_Client_Page_Chat.md` - клиентская страница заказа, чат, хранение доступа 30 дней и сообщения.
4. `Technical_Client_Verification.md` - OTP, доверенные каналы Telegram/Email, срок доверия и блокировка попыток.
5. `Technical_MVP_Implementation_Decisions.md` - технические решения перед реализацией MVP: стек, хранение данных, OTP, QR, клиентский доступ, роли, журналы и платежная ссылка.
6. `Technical_Order_CRM_Workflow.md` - технический поток CRM менеджера, кухня, курьер, статусы и события.
7. `Technical_Order_Data_Model.md` - основные сущности заказа, корзины, позиций, статусов и истории.
8. `Technical_Payment_Delivery.md` - платежная ссылка, наличная оплата, QR-доставка и закрытие заказа.
9. `Technical_Risk_Resolution.md` - желтый/красный список, урегулирование, совпадение признаков и архивы.
