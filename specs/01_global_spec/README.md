# Global Spec

## Назначение раздела

Этот раздел хранит глобальное видение проекта: смысл системы, бизнес-цели, ключевых участников, фундаментальные принципы и границы продукта.

Global Spec отвечает на вопрос: зачем существует проект и какие принципы нельзя потерять на следующих уровнях проработки.

## Правила раздела

1. В разделе хранится только глобальное видение проекта.
2. Global Spec не должен описывать экраны, поля форм, API, базу данных, статусы и техническую реализацию.
3. Если новое решение меняет смысл проекта, роли или фундаментальные принципы, сначала обновляется Global Spec.
4. Документы нижнего уровня не должны противоречить Global Spec.
5. Все изменения Global Spec требуют отдельного осознанного утверждения.

## Текущие документы

1. `Global_Spec.md` - глобальная спецификация проекта.

## Что добавлять дальше

В этот раздел можно добавлять только документы, которые уточняют глобальное видение, но не заменяют Functional Map, Feature Specs или Technical Specs.
