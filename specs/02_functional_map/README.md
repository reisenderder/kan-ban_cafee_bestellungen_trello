# Functional Map

## Назначение раздела

Этот раздел хранит функциональную карту проекта: возможности системы, участников, пользовательские пути, ситуации, MVP-границы и зоны дальнейшей проработки.

Functional Map отвечает на вопрос: какие возможности должна дать система и какие сценарии нужно предусмотреть до детализации отдельных функций.

## Правила раздела

1. Functional Map раскрывает Global Spec через возможности, роли, пользовательские пути и ситуации.
2. Functional Map не должен описывать API, базу данных, фреймворки и финальную техническую реализацию.
3. MVP-блокеры должны быть явно отмечены или закрыты отдельным разделом.
4. Открытые вопросы должны оставаться видимыми и не скрываться внутри текста.
5. Feature Specs должны ссылаться на Functional Map и не менять ее смысл без отдельного утверждения.

## Текущие документы

1. `Functional_Map.md` - функциональная карта проекта.

## Что добавлять дальше

В этот раздел можно добавлять только документы, которые уточняют карту возможностей и сценариев, но не заменяют Feature Specs, Technical Specs или User Stories.
