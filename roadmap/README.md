# Дорожная карта проекта

## Назначение

Этот раздел хранит хронологическую историю фактически завершенных и проверенных работ по основным областям проекта.

Roadmap обновляется после завершения каждого Work Plan. Она не заменяет планы разработки и не содержит будущих обещаний: активная работа остается в `../work_plans/active/`, а roadmap фиксирует только подтвержденный результат.

## Журналы

1. `Data_Roadmap.md` - модели данных, хранение, схемы, миграции, API-контракты данных и связанные механизмы.
2. `Frontend_Design_Roadmap.md` - frontend, интерфейсы, UX, визуальные правила, дизайн-система и доступность.
3. `AI_Analysis_Roadmap.md` - AI-функции, модели, prompts, агенты, AI-анализ, evaluations и защитные ограничения.

## Когда обновлять

Roadmap обновляется только после того, как:

1. реализация соответствующего плана завершена;
2. пользователь вручную протестировал результат;
3. пользователь явно подтвердил, что результат работает;
4. план переносится из `../work_plans/active/` в `../work_plans/completed/`.

Обновление выполняется до `git add`, `commit`, `push` или создания pull request и является обязательной частью завершения плана.

## Правила хронологии

1. Записи располагаются от старых к новым; новая запись добавляется в конец журнала.
2. Дата записывается в формате `YYYY-MM-DD` и означает дату подтверждения завершенного плана пользователем.
3. Одна запись описывает результат одного завершенного плана в одной области.
4. Если план затронул несколько областей, отдельная запись добавляется в каждый соответствующий журнал.
5. Если область не была затронута, пустая запись в ее журнал не добавляется.
6. Каждая запись ссылается на файл в `../work_plans/completed/`.
7. Старые записи не удаляются и не переписываются. Исправление добавляется новой датированной записью.
8. В roadmap фиксируется только фактически реализованное и проверенное поведение.

## Формат записи

```md
## YYYY-MM-DD — Название завершенного плана

- **План**: `../work_plans/completed/Plan_Name.md`
- **Выполнено**: краткий перечень работ в этой области.
- **Результат**: доступное пользователю или системе итоговое состояние.
- **Проверка**: как подтвержден результат.
```
